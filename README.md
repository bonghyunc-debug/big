# 양도소득세 신고서 작성 웹앱 — Claude Opus YOLO 개발 킷

이 폴더는 **“완전한 양도소득세 신고서 작성(별지 제84호 + 부표 1/2/2의2/3) 웹앱”**을 **클로드 오퍼스(Claude Opus)** 에게 *질문 없이* 한 번에 구현시키기 위해 필요한 **스펙/운영절차(SOP)/게이트(커버리지 체크)/시나리오 테스트 프레임**을 제공합니다.

- 운영 형태: **데스크탑(PC) 전용 웹앱**, **로컬 실행**, **업로드 미지원**, **한국어만**
- 핵심 원칙: **판정은 사용자가 하고, 앱은 “서식/법령에 명시된 계산·검증·출력 연계”를 정확히 수행**
- 출력 범위: **신고서(서식/부표/첨부서류 체크리스트) PDF 미리보기 + 단면 인쇄/다운로드**
- 제외: 납부서/납부지 출력, 홈택스 로그인 자동화(자격증명 수집 금지)

## 포함된 주요 파일
- `CLAUDE_OPUS_YOLO.md`: 클로드 오퍼스에 그대로 붙여넣을 **메인 지시문**
- `docs/spec/field_catalog.csv`: 서식 필드 카탈로그(전수)
- `docs/spec/rule_matrix.csv`: 규칙 매트릭스(CALC/VAL/OUT)
- `docs/spec/enum_registry.json`: 코드/드롭다운 단일 출처
- `docs/spec/evidence_log.md`: 근거 로그(국세청/법령 링크, 확인일 포함)
- `docs/spec/SOP.md`: 전수조사 운영절차(현장 체크리스트)
- `scripts/spec/check-spec.mjs`: “필드/규칙 커버리지 게이트” 자동 검사 스크립트
- `docs/tests/SCENARIOS.md`: 시나리오 테스트 설계(대표/경계값)
- `docs/tests/golden/README.md`: 홈택스 골든 PDF 비교(자격증명 공유 없이)

## 사용 방법(권장)
1) 기존 코드베이스(당신의 최신 앱 ZIP 또는 Git repo)를 준비합니다.  
2) 이 킷의 파일/폴더를 **그 코드베이스 루트에 병합**합니다(덮어쓰기 전에 Git 커밋 권장).  
3) 클로드 오퍼스에 `CLAUDE_OPUS_YOLO.md` 내용을 그대로 입력하여 구현을 진행합니다.  
4) 구현 완료 후 로컬에서 다음을 수행해 **완전성**을 확인합니다.
- `npm run spec:check` (필드/규칙 게이트)
- `npm test` (시나리오 테스트)

> 참고: 국세청 서식 다운로드 페이지(별지 제84호 및 부표)와 국가법령정보센터(소득세법/시행령/시행규칙)는 근거의 “Single Source of Truth”로 사용합니다.
